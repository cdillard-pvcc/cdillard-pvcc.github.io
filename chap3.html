Operating Systems Terminology

TERM DEFINITION
operating system software that controls the operation of a computer. It provides the means by which a user can store and retrieve files, provides the interface by which a user can request the execution of programs, and provides the environment necessary to excute the programs requested.
job queue storage of jobs waiting for execution.
FIFO first-in, first-out. Objects are removed from the queue in the order in which they arrived.
interactive processing operating system that allows program being executed to carry on a dialogue with the user through remote terminals.
time-sharing multiprocessing provide service to multiple users at the same time by dividing time into intervals and then execution of each job is restricted to one interval at a time.
application software programs performing tasks particular to the machine's utilization.
utility software category of system software that consists of programming performing activities that are fundamental to computer installations but not included in the operating system.
graphical user interface (GUI) method of communicating with users in which objects to be manipulated, such as files and programs, are represented pictorially on the display as icons.
shell communicating with users through textual messages using a keyboard and monitor screen.
kernal internal part of the operating system that contains software components that perform the basic functions required by the computer installation.
device driver components of the kernal, which are software units that communicate with the controllers to carry out operations on the peripheral devices attached to the machine.
file manager part of the kernal whose job is to coordinate the use of the machine's mass storage facilities.
virtual memory large "fictional" memory space created by paging.
boot strapping how the operating system gets started using a procedure that transfers the operating system from mass storage to main memory.
ROM Read-Only Memory - portion of the computer's main memory where the CPU expects to find its initial program. ROM is constructed with special nonvolatile memory cells. Contents can be read but not altered.
deadlock the condition in which two or more processes are blocked from progressing because each is waiting for a resource that is allocated to another.
spooling holding data for output at a later, more convenient time.
super user (or administrator) person that establishes accounts and has highly priviledged access to the operating system.
multi-factor authentication require more than one piece of evidence to gain access to a computer or system.
